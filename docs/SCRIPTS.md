# Scripts catalog

**Audience:** contributors and agents who need to know what's in `scripts/` and `data-repo/scripts/` without reading every file.

Two script directories:

- **[`scripts/`](../scripts/)** — outer repo (Explorer-facing tooling)
- **[`data-repo/scripts/`](../data-repo/scripts/)** — data submodule (dataset CRUD + migrations)

Most scripts are **one-shots** (idempotent, safe to re-run) and write to disk. A few are pure read tools that print to stdout. None of them push to remote — pushing is always a separate manual step.

Numbers in parentheses are the source file's line count, useful for "how much am I about to read" estimates.

## Patterns

- **PowerShell wide-indent JSON preservation.** Several scripts (`add-tablet`, `find-or-add-pen`, `add-pressure-session`, `apply-pressure-backfill`) write back to files by either shelling out to PowerShell `ConvertTo-Json` or by surgical text splicing. The data files in `data-repo/data/` use a non-standard indent style (29-space record braces, 33-space fields, 49-space tuple brackets, 53-space numbers) that `JSON.stringify` won't reproduce. Don't `JSON.parse → JSON.stringify` round-trip these files; the diff will be enormous.
- **Idempotent / dry-run by default.** Most writers accept `--dry-run` and re-running them with the same input is a no-op. Add this flag to new scripts.
- **Validate after writing.** Most write-scripts print a reminder to run `npm run data-quality` afterwards (it's not automatic so multiple inserts can be batched before validation).
- **Generators emit to `/tmp` first.** `gen-brand-data.example.mjs` (and its copies) start with `OUT_DIR = '/tmp/...'` so the first run can be diffed before promoting to `data-repo/data/`.

## How to add a new script

1. Place it in `scripts/` (Explorer-facing) or `data-repo/scripts/` (dataset CRUD).
2. Start with a file-top docstring that lists: **purpose**, **usage**, **what it writes**, **idempotency**.
3. Use `tsx` for TypeScript (`npx tsx`); pure `.mjs` for Node + JS; `.ps1` only when you need the Windows COM / PowerShell-specific ecosystem (currently only Wacom CDN URLs and `ConvertTo-Json`).
4. Default to `--dry-run` support if it writes anywhere.
5. **Always preserve the wide-indent JSON format** when touching `data-repo/data/*.json` — either shell out to PowerShell `ConvertTo-Json` (see `add-tablet.ts`) or do a surgical text splice (see `apply-pressure-backfill.mjs`).
6. Add a row to this file.
